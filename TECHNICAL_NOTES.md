# ARCHITECTURE

## Задание
Мини-CRM для сбора и обработки заявок с сайта через виджет: создание заявок по API, статистика, админ-панель

## Стек и зависимости
- Laravel 12, PHP 8.4
- MySQL 8.0
- Auth: Laravel Breeze (Blade)
- Роли: spatie/laravel-permission
- Файлы: spatie/laravel-medialibrary
- Frontend сборка: Vite (dev-server в docker)

## Ключевые решения и причины

### Breeze (Blade) для авторизации менеджера
Готовая базовая аутентификация без лишних зависимостей и возможностей. Достаточно для данного задания.

### timestamps в таблицах
`created_at` используется для статистики по периодам (сутки/неделя/месяц)

### Обязательные поля Customer
`name`, `email`, `phone_e164` обязательны, чтобы не создавать безымянных клиентов, которых потом невозможно различать. Упрощает аналитику и валидацию.

### status как строка + PHP Enum
Строки (`new`, `in_progress`, `closed`) проще читать в БД и отлаживать. Другой вариант был `tinyint`.

### cascadeOnDelete для tickets.customer_id
Для упрощения. Ticket не имеет смысла без Customer. В реальном проекте Customer вообще бы не удалялся из БД.  

### Уникальность email/phone и поведение при создании заявки
Для упрощения один клиент - это пара `email` + `phone_e164`. Ситуации, когда клиент поменял email или номер телефона, не рассматриваются. Нет рисков случайной перезаписи данных.

### Трактовка ограничения 1 заявка в сутки
Считаем, что речь о календарном дне (с 00:00 по timezone приложения), а не 24 часа с момента последнего создания

### Нет защиты от двух одновременных заявок
Случай, когда не сработает ограничение 1 заявка в сутки. Для тестового задания это избыточно, маловероятный сценарий.

### Авто-тесты не изолированы
Нет отдельной тестовой базы. Для тестового задания это избыточно.

### Поиск по email только с начала строки
Выигрыш в скорости отработки запроса. Полезность поиска по любой части строки (как в случае с телефоном) маловероятна.

### Локали и тайм зоны
Для упрощения используется тайм зона по умолчанию (UTC) и локаль из .env. Тайм зону UTC удобно хранить в БД, если потребуется работать с пользователями из разных тайм зон.
